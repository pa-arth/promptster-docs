---
title: "Quick Start"
description: "Install, redeem your key, and start your assessment in under 5 minutes."
---

This is the happy path from email to submission. If anything misbehaves along the way, [Troubleshooting](/candidates/troubleshooting) has a fix for every common snag — and `promptster doctor` will diagnose your setup at any point.

<Steps>
  <Step title="Install the CLI">
    ```bash
    curl -fsSL https://get.promptster.ai | sh
    ```

    The script downloads a single binary to `~/.promptster/bin/promptster` and adds it to your shell profile. Restart your shell (or run the `export PATH=...` line the installer prints) so `promptster` is on your `PATH`.

    Prefer npm? `npm install -g @promptster/cli` works too. See [Install](/candidates/cli/install) for details and verification.
  </Step>

  <Step title="Start your assessment">
    Run `promptster start` with the key from your invitation email:

    ```bash
    promptster start PST-XXXX-XXXX
    ```

    `start` validates the key, shows your assessment details (organization and time limit), and walks you through setup. The assessment *title* is intentionally hidden — the task brief is the single source of truth for what you're asked to do.
  </Step>

  <Step title="Review and accept the data notice">
    The CLI shows exactly what is and isn't captured, then asks you to type `yes` to accept the [terms of service](https://promptster.ai/terms).

    **Captured during your session:**

    | What | Example |
    |------|---------|
    | Prompts you send to the AI | "Refactor this function to use async/await" |
    | AI tool calls and responses | File edits, shell commands, web searches |
    | Code changes (unified diffs) | `+12 / -3` in `auth.ts` |
    | Terminal commands and exit codes | `pnpm test` → exit 0 |
    | Which files you read or search | Opened `schema.ts` |

    **Never captured:** keystrokes, screen recordings, your clipboard, environment variables/secrets, or anything outside the workspace.

    You'll then be asked one **optional** question — whether to allow *cadence-based authorship checks* (timing between your prompts and commands, to help reviewers distinguish typed work from pasted work). You can answer `no` and still take the assessment normally. See [Privacy & Data](/candidates/privacy).
  </Step>

  <Step title="Choose your AI tool and workspace">
    `start` asks which AI tool you'll use — **Claude Code, Codex, or Cursor** — and where to work. You can use the current folder, enter a custom path, or let it create a fresh `~/promptster-test` directory. If the assessment ships a starter repo, the CLI clones it at the correct commit for you.

    It then writes your task to `TASK.md` and configures your tool's hooks. In the default managed mode it also routes the AI through the Promptster proxy so you don't sign in to the model yourself; if the assessment uses your own subscription, you'll sign in to your tool as usual. When setup finishes you'll see your task brief and the workspace path.
  </Step>

  <Step title="Open your tool from the workspace — then build">
    <Warning>
      Open your AI tool (Claude Code, Codex, or Cursor) **from inside the workspace directory**, in a new window. Hooks and the AI proxy are scoped to that folder — a window that was already open, or one started from a different directory, will **not** be captured.
    </Warning>

    ```bash
    cd ~/promptster-test   # the workspace path start printed
    claude                 # opens Claude Code here (use codex or cursor for those tools)
    ```

    Now work normally. In the default managed mode you don't need to sign in to the model at all — Promptster routes it through the proxy for the assessment. If the assessment uses your own subscription, just sign in to your tool as you normally would. Check your task or time remaining anytime:

    ```bash
    promptster brief
    ```
  </Step>

  <Step title="Explain your key decisions">
    When you make a meaningful choice — picking an approach, a data structure, a tradeoff — record *why* in your own words. Inside Claude Code, use the slash command:

    ```
    /explain chose idempotency keys over an app-level lock so Stripe retries are safe
    ```

    Or from any terminal in the workspace:

    ```bash
    promptster explain
    ```

    This is one of the most valuable parts of your submission. The CLI will also nudge you after a chunk of work if you haven't explained anything recently. See [Decisions](/candidates/cli/decisions).
  </Step>

  <Step title="Submit when done">
    ```bash
    promptster done
    ```

    This bundles and uploads your workspace, flushes remaining events, marks the session complete, and prints a link to your own session replay (it's copied to your clipboard). If you started decision notes but left them empty, `done` asks you to finish them first — or run `promptster done --auto` to skip that check.

    Once submitted, the workspace is safe to delete and the hiring team is notified automatically.
  </Step>
</Steps>

<Note>
  Your session data is retained for 30 days. To request deletion or a copy of your data, see [Privacy & Data](/candidates/privacy).
</Note>
