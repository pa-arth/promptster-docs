---
title: "Get Interview Follow-ups"
api: "GET /v1/sessions/:id/interview-followups"
description: "Retrieve AI-generated interview follow-up questions based on the session."
---

Returns AI-generated follow-up questions tailored to the candidate's session. These questions probe deeper into the candidate's decisions, approach, and understanding based on what they actually did during the assessment.

Generated by Claude Sonnet as part of the worker pipeline after session completion.

### Authentication

Requires either an org API key or a candidate key.

```
Authorization: Bearer psk_live_...
```

<ParamField path="id" type="string" required>
  The session ID.
</ParamField>

<ResponseField name="kind" type="string">
  Always `interview_followups_v1`.
</ResponseField>

<ResponseField name="interviewFollowups" type="object">
  The follow-ups artifact.
  <Expandable title="properties">
    <ResponseField name="followups" type="array">
      List of follow-up question objects.
      <Expandable title="properties">
        <ResponseField name="question" type="string">The follow-up question to ask the candidate.</ResponseField>
        <ResponseField name="context" type="string">Context from the session that prompted this question.</ResponseField>
        <ResponseField name="focus_area" type="string">The area this question probes (e.g. `architecture`, `debugging`, `testing`).</ResponseField>
        <ResponseField name="difficulty" type="string">Question difficulty: `easy`, `medium`, or `hard`.</ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="createdAt" type="string">
  ISO 8601 timestamp when the follow-ups were generated.
</ResponseField>

<ResponseExample>
```json 200
{
  "kind": "interview_followups_v1",
  "derivationV": 1,
  "interviewFollowups": {
    "followups": [
      {
        "question": "You chose useReducer over useState for the form state. Walk me through how you'd handle async validation in that reducer.",
        "context": "Candidate switched from useState to useReducer at the 25-minute mark after encountering a stale closure bug.",
        "focus_area": "architecture",
        "difficulty": "medium"
      },
      {
        "question": "Your test suite covers the happy path well. How would you approach testing the error boundary you added?",
        "context": "Candidate added an error boundary component but did not write tests for it.",
        "focus_area": "testing",
        "difficulty": "hard"
      }
    ]
  },
  "createdAt": "2026-04-07T10:35:00.000Z"
}
```
</ResponseExample>

<RequestExample>
```bash curl
curl https://api.promptster.ai/v1/sessions/sess_abc123/interview-followups \
  -H "Authorization: Bearer psk_live_abc123def456"
```
</RequestExample>
