---
title: "List Candidate Keys"
api: "GET /v1/assessments/:id/keys"
description: "List all candidate keys for an assessment."
---

Returns all active (non-deleted) candidate keys for the specified assessment, ordered by creation date. Includes the total token cost for each candidate's session.

### Authentication

Requires an org API key.

```
Authorization: Bearer psk_live_...
```

<ParamField path="id" type="string" required>
  The assessment ID (UUID).
</ParamField>

<ResponseField name="keys" type="array">
  List of candidate key objects with usage cost.
  <Expandable title="properties">
    <ResponseField name="id" type="string">Key record ID (UUID).</ResponseField>
    <ResponseField name="key" type="string">The candidate key (e.g. `PST-ABCD-1234`).</ResponseField>
    <ResponseField name="candidateEmail" type="string | null">Candidate's email.</ResponseField>
    <ResponseField name="candidateName" type="string | null">Candidate's name.</ResponseField>
    <ResponseField name="status" type="string">`pending`, `active`, `completed`, or `expired`.</ResponseField>
    <ResponseField name="sessionId" type="string | null">Linked session ID once the candidate starts.</ResponseField>
    <ResponseField name="expiresAt" type="string">ISO 8601 expiration timestamp.</ResponseField>
    <ResponseField name="redeemedAt" type="string | null">When the candidate redeemed the key.</ResponseField>
    <ResponseField name="completedAt" type="string | null">When the candidate submitted.</ResponseField>
    <ResponseField name="totalCostUsd" type="number">Total LLM token cost in USD for this candidate's session.</ResponseField>
  </Expandable>
</ResponseField>
