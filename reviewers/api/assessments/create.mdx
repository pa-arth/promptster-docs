---
title: "Create Assessment"
api: "POST /v1/assessments"
description: "Create a new hiring assessment for your organization."
---

Create an assessment that defines the coding task candidates will complete. You can link it to an OSS issue from the Promptster problem library, or provide your own task brief and repo URL.

### Authentication

Requires an org API key with an active subscription.

```
Authorization: Bearer psk_live_...
```

<ParamField body="title" type="string" required>
  The assessment title shown to candidates and in the dashboard.
</ParamField>

<ParamField body="role" type="string" required>
  The role this assessment is for (e.g. "Senior Frontend Engineer").
</ParamField>

<ParamField body="taskBrief" type="string">
  Markdown description of the task. Required if `issueId` is not provided. Auto-populated from the issue if omitted.
</ParamField>

<ParamField body="repoUrl" type="string">
  URL of the repository candidates will work in. Must be a valid URL.
</ParamField>

<ParamField body="issueId" type="string">
  ID of an OSS issue from the Promptster problem library. When provided, the task brief is auto-populated from the issue.
</ParamField>

<ParamField body="timeLimitMinutes" type="integer">
  Time limit for the assessment in minutes. Must be a positive integer.
</ParamField>

<ParamField body="expiresInDays" type="integer" default="7">
  Number of days before candidate keys expire. Must be a positive integer.
</ParamField>

<ParamField body="tokenBudgetUsd" type="number">
  Maximum token spend allowed per candidate in USD. Must be positive and at most 1000. Claude Code and Codex draw from this **single shared** per-candidate budget — allowing both tools does not double it.
</ParamField>

<ParamField body="allowedTools" type="string[]" default='["claude","codex","cursor"]'>
  Which AI coding tools the candidate may use. One or more of:

  - `claude` — Claude Code. Requires an org **Anthropic** key.
  - `codex` — Codex CLI. Requires an org **OpenAI** key.
  - `cursor` — Cursor. The candidate brings their **own** Cursor subscription, so no org key is needed.

  If you **omit** this field, it is auto-derived from the provider keys your org has configured: `claude` is included if a usable Anthropic key resolves, `codex` if a usable OpenAI key resolves, and `cursor` is always included.

  If you **set** it explicitly, your org must have a usable key for each selected tool's provider (`claude` → Anthropic, `codex` → OpenAI; `cursor` needs none), otherwise creation fails with `400 { "code": "byok_required" }`.
</ParamField>

<ParamField body="anthropicKeyMode" type="string" default="org_default">
  Which Anthropic key the assessment uses when `claude` is allowed. `org_default` uses your org's default Anthropic key; `byok` uses the specific key named by `anthropicKeyId`.
</ParamField>

<ParamField body="anthropicKeyId" type="string">
  ID (UUID) of the Anthropic key to use when `anthropicKeyMode` is `byok`. Discover key IDs via [List Provider Keys](/reviewers/api/keys/list). An invalid or unknown ID fails with `400 { "code": "byok_key_invalid" }`.
</ParamField>

<ParamField body="openaiKeyMode" type="string" default="org_default">
  Which OpenAI key the assessment uses when `codex` is allowed. `org_default` uses your org's default OpenAI key; `byok` uses the specific key named by `openaiKeyId`.
</ParamField>

<ParamField body="openaiKeyId" type="string">
  ID (UUID) of the OpenAI key to use when `openaiKeyMode` is `byok`. Discover key IDs via [List Provider Keys](/reviewers/api/keys/list). An invalid or unknown ID fails with `400 { "code": "byok_key_invalid" }`.
</ParamField>

<Note>
  To select a specific provider key, first call [List Provider Keys](/reviewers/api/keys/list) to find its `id`, then pass it as `anthropicKeyId` or `openaiKeyId`.
</Note>

<ResponseField name="assessment" type="object">
  The created assessment object.
  <Expandable title="properties">
    <ResponseField name="id" type="string">Unique assessment ID (UUID).</ResponseField>
    <ResponseField name="orgId" type="string">Your organization ID.</ResponseField>
    <ResponseField name="title" type="string">Assessment title.</ResponseField>
    <ResponseField name="role" type="string">Target role.</ResponseField>
    <ResponseField name="taskBrief" type="string">Task description.</ResponseField>
    <ResponseField name="repoUrl" type="string | null">Repository URL.</ResponseField>
    <ResponseField name="issueId" type="string | null">Linked OSS issue ID.</ResponseField>
    <ResponseField name="timeLimitMinutes" type="integer | null">Time limit in minutes.</ResponseField>
    <ResponseField name="tokenBudgetUsd" type="number | null">Token budget in USD.</ResponseField>
    <ResponseField name="allowedTools" type="string[]">The resolved set of tools the candidate may use (`claude`, `codex`, `cursor`).</ResponseField>
    <ResponseField name="expiresInDays" type="integer">Key expiry period in days.</ResponseField>
    <ResponseField name="createdAt" type="string">ISO 8601 timestamp.</ResponseField>
  </Expandable>
</ResponseField>
