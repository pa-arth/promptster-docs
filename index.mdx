Promptster runs alongside Claude Code and Cursor during a coding assessment, capturing a record of the session: prompts, file changes, commands, test runs, and the candidate's own decision rationale. Hiring teams get an evidence trail to evaluate engineering judgment — not just final output.

## How it works

<Steps>
  <Step title="Hiring team creates an assessment">
    Define the task brief, role, and time limit. Generate candidate keys to distribute.
  </Step>
  <Step title="Candidate redeems their key">
    `promptster start PST-XXXX-XXXX` sets up the workspace and configures Claude Code or Cursor to capture the session — with the AI routed through Promptster, so the hiring team covers the cost.
  </Step>
  <Step title="Candidate completes the task">
    They work normally. Promptster captures prompts, file diffs, commands, and the decisions they explain with `/explain`.
  </Step>
  <Step title="Candidate submits">
    `promptster done` uploads the workspace, closes the session, and returns a private replay link.
  </Step>
  <Step title="Hiring team reviews">
    Browse the session timeline, read the candidate's decisions, and examine derived signals to evaluate engineering judgment.
  </Step>
</Steps>

<Note>
  Promptster only captures activity inside the assessment workspace and AI coding tool. Keystrokes, screen contents, clipboard, browser activity, and environment variables are never recorded. Session data is retained for 30 days. See [Privacy & Data](/candidates/privacy).
</Note>
